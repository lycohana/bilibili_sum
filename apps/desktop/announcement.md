# 最近更新了不少好用的东西

这段时间 BiliSum 不只是修了一些问题，也补上了很多日常会用到的能力：本地视频、多 P 视频、知识库、笔记导出、B 站登录风控处理，还有更快的启动速度。这里挑最值得你知道的说。

## 1. 打开应用更快了

Windows 端启动慢的问题做了一轮重点优化。现在点开应用后，会先把界面显示出来，转写、GPU、知识库这些比较重的能力会在后台继续准备。

- 不用再盯着空窗口等半天。
- 标题栏会显示运行状态，能看出服务、运行时和配置是否准备好。
- 上次没跑完的任务也会自动恢复，不容易因为重启中断。

## 2. 视频处理能力更完整了

现在可以处理的不只是单个公开视频，更多真实使用场景也补上了。

- 支持导入本地视频，并在应用里播放和总结。
- 支持多 P 视频批量生成摘要，也可以批量重新生成。
- 新增多 P 视频全集总结，更适合课程、合集、长系列内容。
- 视频库增加分页、收藏和更清晰的最近视频展示，视频多了也更好找。

## 3. 看完之后可以沉淀到知识库

BiliSum 现在可以把已经总结过的视频沉淀成本地知识库，不只是“看完生成一份摘要”。

- 新增知识库入口，可以检索视频摘要、片段内容和知识笔记。
- 支持对知识库提问，比如“之前哪个视频讲过这个概念？”。
- 支持标签和关系网络，方便按主题管理内容。
- 旧视频也可以重建索引，重新纳入检索和问答。

## 4. 笔记和导出更适合长期整理

总结结果现在更适合放进自己的笔记系统里继续整理。

- 新增知识笔记，内容会围绕核心结论、关键要点和可复用知识整理。
- 支持导出 Markdown / Obsidian 风格笔记。
- 思维导图和知识笔记链路也做了适配，适合复习和二次整理。

## 5. B 站解析遇到风控时更好处理

如果批量解析 B 站视频时遇到登录态、Cookie 或风控问题，现在有更直接的解决方式。

- 新增内置 B 站登录窗口，可以扫码或账号登录。
- 登录成功后，应用会自动保存可用的登录态。
- 遇到 HTTP 412 等问题时，提示会更明确，不再只给一串看不懂的错误。

## 6. 设置和使用体验也顺手补了一轮

- 新增设置搜索，配置项多了以后也能快速定位。
- 支持自定义总结、知识笔记等提示词。
- 恢复默认提示词更稳，不容易出现前后端默认值不一致。
- 外部链接、文件清理和桌面窗口行为做了安全和稳定性修复。

总之，这几版主要是在补“真正长期用起来会需要的东西”：启动更快，视频来源更多，结果能沉淀，出问题时也更容易处理。
