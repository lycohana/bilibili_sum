# 重大更新！！

这次更新重点补齐了「看完之后怎么沉淀」和「B 站批量解析更稳」两件事。上次发版之后的新能力，主要都集中在下面这些地方：

## 1. 知识库正式上线

BiliSum 现在不只是帮你总结视频，还可以把已经处理过的视频沉淀成自己的本地知识库。

- 新增「知识库」入口，可以对本地视频摘要、知识笔记和片段内容进行检索。
- 支持 AI 知识库问答：你可以直接问“之前哪个视频讲过这个概念？”、“帮我整理某个主题下的观点”，应用会结合本地内容回答。
- 新增标签体系：可以给视频打标签，也支持自动打标，方便按主题管理内容。
- 新增标签关系网络和知识库工作台，可以更直观地查看视频、标签和主题之间的关联。
- 支持重建知识库索引，旧视频也可以重新纳入检索和问答。

## 2. 新增知识笔记与 Obsidian / Markdown 导出

每个总结结果现在会额外生成一份更适合复习和二次整理的知识笔记。

- 新增 `knowledge_note.md`，内容会围绕核心结论、关键要点和可复用知识整理。
- 支持导出 Markdown / Obsidian 风格笔记，方便直接放进自己的笔记库。
- 思维导图和知识笔记链路也做了适配，后续更适合作为长期知识管理入口。

## 3. 批量解析遇到 B 站风控时更好处理了

之前批量解析 B 站视频时，可能会遇到 HTTP 412、登录态失效、浏览器 Cookie 读取失败等问题。这次加入了更直接的处理方式。

- 新增 BiliSum 内置的 B 站登录窗口。
- 遇到风控或登录态问题时，可以直接扫码 / 账号登录 B 站。
- 登录成功后，应用会自动捕获 B 站登录态，并保存成 yt-dlp 可用的 `cookies.txt`。
- 保存成功后重新提交任务即可，不再强依赖读取 Chrome / Edge / Firefox 的本机 Cookie 数据库。
- 错误提示也更明确了：如果仍然遇到 412，会提示切换网络/IP、稍后重试或手动导出 cookies。

## 4. 其他优化

- 完善知识库运行时依赖加载与同步链路，减少本地环境缺依赖导致的不可用情况。
- 优化知识库问答 Agent 链路和前端交互，回答过程、来源和工具调用展示更清晰。
- 补充了知识库、导出、设置和运行时相关测试，让这次大功能更稳一点。

祝你的视频库真的变成知识库，而不是另一个“以后再看”的黑洞。
