# v1.16.0 更新说明

## 全新：VLM 理解型图文笔记

和之前的"插图笔记"（在原文里插几张截图）完全不同，这是一版**从零重构**的图文笔记生成方式：

- **画面驱动写作**：VLM 阅读原始笔记和每张截图的客观信息，以画面为线索重新组织一篇全新的笔记。段落结构、表达方式都会被重新编排，不再是原文+图片点缀。
- **数据源头重构**：帧描述从叙述性文本改为客观事实列表（`key_facts`），从源头杜绝"画面呈现了...""该图展示了..."等流水账句式。
- **精选配图分散插入**：只选最重要的 3-6 张图，每张紧跟对应知识点段落，像专业教材一样段落与图片交替呈现，禁止堆积在末尾。
- **合成稳定性**：预过滤低质量帧，合成超时提升到 300s，大视频不易回退。

在设置页 → 图文笔记形式中选择「VLM 理解型图文笔记」即可启用。

> **使用提示**：VLM 理解型笔记会调用多模态模型处理每一张截图，API 费用高于纯文本模式。如果希望控制成本，可以在设置中**减少截取图片数量**（`visual_evidence_max_frames`）或增大截图间隔（`visual_evidence_frame_interval_seconds`）。

> **已知限制**：超长视频（1 小时以上）生成的图文笔记可能较为简略，因为当前 VLM 合成需要在单次请求中处理完整笔记。视频截图逻辑也在持续优化中——设置内可以**自定义捕获帧规划的提示词**（`visual_frame_planning_prompt`），欢迎根据自己的视频类型调整。如果你有更好的截图策略或提示词方案，欢迎参与贡献！

## 全新：视觉 VLM 多提供商支持

画面理解和图文笔记合成现在支持**独立配置视觉模型**，不再强制跟随主 LLM。设置页可单独设置视觉模型的提供商、Base URL、API Key 和模型名称，与主 LLM 完全解耦。

- **多提供商**：支持 OpenAI、Anthropic、OpenAI 兼容接口及自定义端点。选中后图片格式、请求端点、认证头、响应解析全部自动适配。
- **智能格式切换**：第三方 Anthropic 兼容端点的图片请求会自动回退 OpenAI 格式，避免因端点不支持原生图片协议导致超时。
- **独立连接测试**：可单独测试视觉模型的图片识别和 JSON 输出能力。

## 全新：桌面端体验重构

- **动画启动画面**：启动时显示品牌启动画面，加载完成后平滑过渡到主界面。
- **应用内更新**：设置页可直接检查并安装新版本，无需手动下载安装包。
- **整体 UI 重构**：侧边栏导航、设置页布局、首页最近视频列表全面优化。

## 全新：macOS 支持

桌面端现已支持 macOS 构建和打包，macOS 用户可直接使用 `.dmg` 安装。

## ASR 转写增强

- **多模态 ASR**：支持 OpenAI 兼容的多模态音频模型（如 mimo-v2-omni）进行语音转文字，长音频自动切片 + 重试，空响应自动回退 `reasoning_content`。
- **SiliconFlow 长音频**：突破单次 60 分钟限制，自动 ffmpeg 切片 + 并发识别，1 小时以上视频也能正常转写。
- **切片参数可调**：多模态 ASR 和 SiliconFlow ASR 的切片时长、重试次数、并发数均可在设置页调整。
- **CUDA 安装体验增强**：转录诊断功能完善，GPU 渠道运行时同步问题修复。
